# Implementation Plan: M03.1 — Protocol Revision: Copy-drop + Return-value Bridge

**Branch**: `006-m03-1-protocol-revision` | **Date**: 2026-05-21 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/006-m03-1-protocol-revision/spec.md`

## Summary

Revise the M03 event protocol on three points: (1) gate `SlotDrop` emission on the slot's binding type — only non-Copy slots emit drops; (2) add a `MemEvent::ReturnValue { frame_id, value, span }` variant emitted between body completion and `FrameLeave`; (3) drop the redundant `FrameEnter.params` field. Update M04's `Cursor` to consume the new variant and extend `StateSnapshot` with an optional `pending_return: Option<PendingReturnView>` field that the JS renderer paints as a transient annotation on the closing frame's card. Re-snapshot M03's integration tests and regenerate M04's pre-recorded traces.

Authority chain: `MILESTONES.md` › M03.1 → `spec.md` (this feature) → this plan.

## Technical Context

**Language/Version**: Rust 2024 edition (same toolchain as M03/M04). No new toolchain requirements.
**Primary Dependencies**: existing `indexmap`, `serde`, `serde_json`, `wasm-bindgen`, `js-sys`, `console_error_panic_hook`. **No new deps** — this is a protocol revision.
**Storage**: in-memory; trace JSON shape gains the new `ReturnValue` variant case and loses the `FrameEnter.params` field.
**Testing**: existing `cargo test --test m03` updated snapshots (FR-006); `cargo test --test m01` / `m02` byte-identical (SC-002); new Cursor unit test for `ReturnValue` handling (FR-005); manual M04 QA per the SC-008 procedure (SC-006).
**Target Platform**: same as M03/M04 (host + `wasm32-unknown-unknown`).
**Project Type**: Rust library revision + companion M04 UI update.
**Performance Goals**: M03.1 doesn't add hot paths. Cursor's state-at-N replay cost grows by O(1) per `ReturnValue` event (one extra branch in `apply_event`); negligible.
**Constraints**: M01/M02 snapshots untouched (SC-002 byte-identical); M03 snapshots regenerated with predictable diff (SC-004 event-count math); M04 WASM bundle size growth ≤ 5% (SC-005); zero warnings under `-D warnings` (SC-007); M04 manual QA passes (SC-006).
**Scale/Scope**: ~5 files modified (typeck.rs, event.rs, eval.rs, ui.rs, web assets) + 8 M03 snapshots re-baselined + 4 trace JSONs regenerated. Estimated ~200–300 LOC of net code changes. Sizing: M.
