# Implementation Plan: M04 — UI Shell + Replay Cursor

**Branch**: `005-m04-ui-shell` | **Date**: 2026-05-21 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/005-m04-ui-shell/spec.md`

## Summary

Add a `src/ui.rs` module containing a deterministic replay `Cursor` (state-at-step-N computation) and `wasm-bindgen` exports for a browser player. Add a `src/bin/gen_traces.rs` binary that runs the M01 → M02 → M03 pipeline on selected sample programs and writes pre-recorded traces as JSON to `web/traces/`. Add a `web/` directory holding `index.html`, `index.js` (vanilla JS — CodeMirror 6 via ESM CDN), and `style.css`. Build + serve with `trunk`. The page is the project's first end-user artifact: Editor (CodeMirror, read-only, span-decorated) | Stacks (vertical frame cards) | Heap (reserved placeholder) | toolbar (Rewind / Step Back / Play-Pause / Step Forward / step indicator).

Authority chain: `MILESTONES.md` › M04 → `spec.md` (this feature) → this plan. The plan commits to the three spec-deferred decisions (editor framework, UI approach, trace format) plus build tooling.

## Technical Context

**Language/Version**: Rust 2024 edition (same toolchain as M01–M03). `wasm32-unknown-unknown` target for the WASM build. Modern browser JS (ES modules, fetch, async/await).
**Primary Dependencies**: existing `indexmap` regular dep; NEW regular deps `serde` + `serde_json` (trace JSON serde) and `wasm-bindgen` + `js-sys` + `console_error_panic_hook` (WASM bindings + dev ergonomics). All WASM-portable and standard.
**Storage**: pre-recorded traces are static JSON assets under `web/traces/` (gitignored, regenerated by `cargo run --bin gen_traces`).
**Testing**: Rust unit tests in `src/ui.rs` for the `Cursor` state-at-N logic (deterministic, table-driven). Existing `cargo test --test m01/m02/m03` regression. Manual visual verification per a documented procedure in `quickstart.md` (no automated browser tests per spec).
**Target Platform**: modern desktop browsers (Chromium ≥ 110, Firefox ≥ 110, Safari ≥ 16) at viewport ≥ 1024 px. Rust toolchain plus `cargo install trunk` for the dev server.
**Project Type**: Rust library (`rlib`) + WASM bindings (`cdylib`) — same crate, dual `crate-type`.
**Performance Goals**: page loads to first-interactive in < 3 s on broadband; WASM bundle ≤ 2 MB gzipped (SC-005). State-at-N computation is O(N) per cursor move which is fine for L1 traces (typically < 100 events).
**Constraints**: zero new transitive deps beyond `serde`/`serde_json`/`wasm-bindgen`/`js-sys`/`console_error_panic_hook`; CodeMirror loaded via ESM CDN (no bundler step); single-file `index.js`; deterministic cursor; M01/M02/M03 tests untouched (SC-006).
**Scale/Scope**: ≥ 3 pre-recorded samples (we'll ship 4: `arithmetic`, `fn_call`, `shadow`, `div_by_zero`); estimated ~600–900 LOC across `src/ui.rs` + `src/bin/gen_traces.rs` + the web assets.
